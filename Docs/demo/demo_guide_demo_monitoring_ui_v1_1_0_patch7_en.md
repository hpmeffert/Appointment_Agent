# Demo Guide Demo Monitoring UI v1.1.0 Patch 7

Version: v1.1.0-patch7
Audience: Demo
Language: EN

## Best live story for Patch 7

Tell people:

- on the left you see the customer conversation
- in the middle you see what the system offers
- in Monitoring you see the technical result of each click

## Story 1: Reminder with direct action buttons

1. Open the Dashboard.
2. Pick `Appointment Reminder Keep` or `Appointment Reminder Reschedule`.
3. Show the reply buttons.
4. Click `Keep` or `Reschedule`.
5. Then switch to `Monitoring` and show the new event.

What to say:

- This is not just text.
- The buttons represent the reply actions a customer could send in RCS or SMS.
- Every click changes the visible state and the technical trail.

## Story 2: Real slot selection

1. Choose `Standard Booking` or `Appointment Reminder Reschedule`.
2. Advance until slot buttons are visible.
3. Click a real slot label.
4. Show how the story moves forward.

What to say:

- We do not show fake `Slot 1`, `Slot 2`, `Slot 3`.
- We show real labels that a customer can understand immediately.

## Story 3: Google-backed demo preparation

1. Open `Google Demo Control`.
2. Decide between `Simulation` and `Test`.
3. Set `From Date`, `To Date`, and `Appointment Type`.
4. Use `Prepare Demo Calendar` first if you only want a preview.
5. Use `Generate Demo Appointments` if you want real entries in the configured Google test calendar.
6. Verify the entries in Google Calendar.
7. Use `Delete Demo Appointments` or `Reset Demo Calendar` after the demo.

## Important explanation for the audience

- `Prepare Demo Calendar`
  Preview only. No real Google entries are written.

- `Generate Demo Appointments`
  Creates real entries only in `Test` mode.

- `Delete Demo Appointments`
  Removes the generated demo entries again.

- `Reset Demo Calendar`
  Cleans and rebuilds the demo state.
