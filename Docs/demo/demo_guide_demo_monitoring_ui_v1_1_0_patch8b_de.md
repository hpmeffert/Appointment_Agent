# Demo Guide Demo Monitoring UI v1.1.0 Patch 8b

Version: v1.1.0-patch8b
Audience: Demo
Language: DE

## Interaktiver Demo-Ablauf Schritt fuer Schritt

1. „Hier sieht der Kunde eine Erinnerungsnachricht.“
2. „Jetzt klicke ich auf Verschieben.“
3. „Das System laedt die naechsten verfuegbaren Termin-Slots.“
4. „Jetzt waehle ich einen echten Zeitslot aus.“
5. „Das System reserviert diesen Slot temporaer.“
6. „Vor der finalen Buchung prueft es Google Kalender noch einmal live.“
7. „Wenn der Slot noch frei ist, wird gebucht.“
8. „Wenn nicht, schlaegt das System Alternativen vor.“

## Aha-Saetze

- „Das ist nicht nur eine Chatbot-Nachricht. Der Klick aendert wirklich den Buchungsablauf.“
- „Dem Slot wird nicht blind vertraut. Er wird temporaer reserviert und dann noch einmal live geprueft.“
- „Genau so kann das System sichere Parallelbuchungen handhaben.“
- „Wenn eine andere Person den Slot zuerst nimmt, bricht das System nicht auseinander. Es reagiert und zeigt Alternativen.“

## Demo-Vorbereitung

1. Nutze `Simulation`, wenn du ohne echte Google-Schreibzugriffe zeigen willst.
2. Nutze `Test`, wenn du echtes Verhalten im Google-Testkalender zeigen willst.
3. Verwende Google Demo Control fuer Generate oder Reset.
4. Nach der Demo raeumst du mit `Delete` oder `Reset` wieder auf.
