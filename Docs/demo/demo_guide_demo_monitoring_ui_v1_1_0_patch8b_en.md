# Demo Guide Demo Monitoring UI v1.1.0 Patch 8b

Version: v1.1.0-patch8b
Audience: Demo
Language: EN

## Step-by-step interactive demo flow

1. “Here the customer sees a reminder message.”
2. “Now I click Reschedule.”
3. “The system loads the next available appointment slots.”
4. “Now I select a real time slot.”
5. “The system temporarily reserves that slot.”
6. “Before final booking, it checks Google Calendar live once more.”
7. “If the slot is still free, it is booked.”
8. “If not, the system proposes alternatives.”

## Aha sentences

- “This is not just a chatbot message. The click actually changes the booking flow.”
- “The slot is not blindly trusted. It is temporarily reserved and then checked live again.”
- “That is why the system can safely handle parallel booking attempts.”
- “If another person takes the slot first, the system does not break. It reacts and offers alternatives.”

## Demo preparation

1. Switch to `Simulation` if you want a safe demo without Google writes.
2. Switch to `Test` if you want real Google test-calendar behavior.
3. Use Google Demo Control to generate or reset demo appointments.
4. After the demo, clean the calendar with `Delete` or `Reset`.
