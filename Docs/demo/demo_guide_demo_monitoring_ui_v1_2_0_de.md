# Demo Guide Demo Monitoring UI v1.2.0

Version: v1.2.0
Audience: Demo
Language: DE

## Praesentationsablauf

1. „Hier sieht der Kunde eine Erinnerungsnachricht.“
2. „Jetzt klicke ich auf Verschieben.“
3. „Das System laedt echte naechste Slots aus dem Booking-Backend.“
4. „Jetzt klicke ich einen Slot.“
5. „Der Slot ist jetzt temporaer fuer diesen Journey-Ablauf reserviert.“
6. „Jetzt klicke ich auf Bestaetigen.“
7. „Vor der Buchung prueft das System Google Kalender noch einmal live.“
8. „Wenn der Slot noch frei ist, wird erfolgreich gebucht.“
9. „Wenn nicht, zeigt das System Alternativen statt still zu scheitern.“

## Aha-Saetze

- „Das ist nicht nur UI. Der Klick steuert wirklich die Buchungslogik.“
- „Der Slot wird temporaer reserviert und danach noch einmal live geprueft.“
- „Genau so kann das System Parallelbuchungen sicher behandeln.“

## Demo-Vorbereitung

1. Oeffne `/ui/demo-monitoring/v1.2.0`
2. Waehle `Simulation` oder `Test`
3. Nutze Google Demo Control fuer Generate oder Reset
4. Nach der Demo den Testkalender wieder aufraeumen
