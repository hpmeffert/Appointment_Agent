# Demo Guide Demo Monitoring UI v1.2.1

Version: v1.2.1
Sprache: DE

## Demo-Ziel

Zeige, dass die Plattform jetzt mehr ist als nur eine Buchungsdemo.

Die neue Story ist:
- Terminlogik
- Nachrichtenverkehr
- Operator-Sichtbarkeit

## Drei Story-Szenarien

1. Buchungseinladung ueber RCS
2. Reminder-Fallback ueber SMS
3. Operator-Ueberblick im Message Monitor und in den Berichten

## Starker Satz fuer die Demo

„Das ist nicht mehr nur Termin-Automatisierung. Das ist der Anfang einer Messaging Conversation Platform.“

## Wichtige Parameter fuer die Demo

- `channel`
- `direction`
- `status`
- `journey_id`
- `booking_reference`
