# User Guide Demo Monitoring UI v1.1.0 Patch 5

Version: v1.1.0-patch5
Audience: User
Language: DE

## Was du oeffnest

Oeffne das aktuelle Cockpit hier:

- `http://localhost:8080/ui/demo-monitoring/v1.1.0-patch5`

## Was das Kopfmenue bedeutet

- `Dashboard`
  Nutze das fuer die einfache Business-Erklaerung der Story.

- `Monitoring`
  Nutze das fuer technische Events, Traces und Performance-Daten.

- `Einstellungen`
  Nutze das, wenn du die wichtigsten Laufzeitwerte verstehen willst.

- `Google Demo Control`
  Nutze das, wenn du Google Demo-Termine vorbereiten, erzeugen, loeschen oder zuruecksetzen willst.

- `Hilfe`
  Nutze das fuer die Leitfaeden.

## Sehr wichtige Google-Regel

- `Simulation`
  Dabei werden keine echten Google Kalendereintraege erstellt.

- `Test`
  Dabei kann der real konfigurierte Google Testkalender veraendert werden.

## Wichtige Google-Parameter

- `mode`
  Sichere Vorschau oder echte Test-Schreibzugriffe.

- `timeframe`
  Tag, Woche oder Monat.

- `count`
  Wie viele Demo-Termine das System bearbeiten soll.

- `vertical`
  Welches Business-Beispiel fuer die Demo-Daten genutzt wird.

## Sicherer erster Demo-Ablauf

1. Oeffne `Dashboard`.
2. Waehle ein Szenario.
3. Wechsle zu `Google Demo Control`.
4. Bleibe in `Simulation`.
5. Nutze `Prepare Demo Calendar`.
6. Wechsle nur dann zu `Test`, wenn du bewusst echte Schreibzugriffe im Google Testkalender willst.
