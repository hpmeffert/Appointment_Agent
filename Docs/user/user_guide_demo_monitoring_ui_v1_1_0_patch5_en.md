# User Guide Demo Monitoring UI v1.1.0 Patch 5

Version: v1.1.0-patch5
Audience: User
Language: EN

## What you open

Open the current cockpit here:

- `http://localhost:8080/ui/demo-monitoring/v1.1.0-patch5`

## What the top menu means

- `Dashboard`
  Use this when you want to tell the story in a simple business way.

- `Monitoring`
  Use this when you want to see technical events, traces, and performance data.

- `Settings`
  Use this when you want to understand which runtime values shape the demo.

- `Google Demo Control`
  Use this when you want to prepare, create, delete, or reset Google demo appointments.

- `Help`
  Use this when you want the guides.

## Very important Google rule

- `Simulation`
  This does not create real Google Calendar entries.

- `Test`
  This can change the real configured Google test calendar.

## Main Google parameters

- `mode`
  Safe preview or real test writes.

- `timeframe`
  Day, week, or month.

- `count`
  How many demo appointments the system should work with.

- `vertical`
  Which business example should be used for the demo data.

## Good safe first demo flow

1. Open `Dashboard`.
2. Pick a scenario.
3. Switch to `Google Demo Control`.
4. Stay in `Simulation`.
5. Use `Prepare Demo Calendar`.
6. Only switch to `Test` if you intentionally want real Google test-calendar writes.
